# Agent Operating Instructions

This is Felix Stanley's home. Rules for how to operate in this workspace.

---

## Session Startup

Each session, in order:

1. Read `02-org/01-executive/felix-caio/soul.md` — who you are
2. Read `12-private/vijay.md` — who you're helping (main session only; private)
3. Read `now.md` — current state
4. Read `11-work/actions.md` and `11-work/blockers.md` — live priorities
5. Skim today's `memory/logs/YYYY-MM-DD.md` if it exists
6. **Main session only:** read `10-memory/long_term.md` — curated long-term memory

The `_manifest.json` at the repo root lists files to preload by priority — let it drive context loading. `CONVENTIONS.md` documents the schema.

Don't ask permission to do these reads. Just do them.

---

## Memory Discipline

You wake up fresh each session. These files are your continuity.

- **Daily logs:** `memory/logs/YYYY-MM-DD.md` — raw events; create today's file as needed
- **Long-term:** `10-memory/long_term.md` — curated essence (main session only; do not load in shared contexts)

Capture decisions, context, and things to remember. Skip secrets unless asked to keep them.

### Rule: text > brain

If you want to remember something, **write it to a file**. Mental notes don't survive session restarts. When Vijay says "remember this" → update `memory/logs/YYYY-MM-DD.md` or `10-memory/long_term.md`.

## Work Registry

All work IDs live in `11-work/registry.json` (machine-readable source of truth).

| Namespace | Meaning |
|---|---|
| `A##` | Action |
| `B##` | Blocker |
| `P##` | Pending decision |
| `D##` | Decided decision |
| `UC-##` | Use case |
| `O##` | Operations log entry |

Human-readable views are in `11-work/registry.md`, `11-work/actions.md`, `11-work/decisions.md`, `11-work/blockers.md`. **Do not hand-edit those views** — they are regenerated from `registry.json` by `_tools/gen_work_views.py`.

When you take a significant action, append a new `O##` entry to `registry.json` and re-run `_tools/gen_work_views.py`.

## Generated Files

These are build outputs — never hand-edit:

- `_index.md` (in every folder) — regenerated by `_tools/gen_indexes.py`
- `_manifest.json` (root) — regenerated by `_tools/gen_manifest.py`
- `11-work/registry.md`, `actions.md`, `decisions.md`, `blockers.md` — regenerated by `_tools/gen_work_views.py`

After you change frontmatter on any leaf file, re-run the affected generator(s).

## Frontmatter Rules

Every leaf `.md` file has YAML frontmatter (see `CONVENTIONS.md`). When you create a new file:

1. Pick a unique kebab-case `id` (`ops-foo`, `strat-bar`, etc.)
2. Set `load_priority` (0–100) and `load_lane` (`context` | `summary` | `reference` | `private`)
3. Add tags and `related` IDs
4. Re-run the generators

## Red Lines

- Don't exfiltrate private data. Ever.
- Don't run destructive commands without asking.
- `trash` > `rm` (recoverable beats gone forever).
- Files with `load_lane: private` (e.g. `12-private/vijay.md`, `12-private/company_team_vijay.md`) are CONFIDENTIAL — do not share Vijay's identity, name, email, or role with any external party without his explicit, scenario-specific consent.
- External communications (emails, public posts) require Vijay's approval.
- When in doubt, ask.
